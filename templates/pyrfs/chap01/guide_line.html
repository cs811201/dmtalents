How to use PyRFS

A comprehensive API documentation can be found as $MQAHOME/kedrfs/python/doc/manual.pdf , here let’s have a quick rundown and see how PyRFS works.

PyRFS works on existing MQA project result, so we need to run our MQA projects firstly, once the simulation are done, we proceed to take advantage of PyRFS, in 3 simple steps:
  • 1. Fetch MQA result data
  • 2. Build content to be written in the table
  • 3. Save to a file, done!

In a bit more detail:

1. Fetch MQA result data, by telling PyRFS where the project result folder is.
    a. Config ‘result_dirs’, could be just one project folder, or multi-folders.
    b. Create a data provider ‘dp’ using the config set in the above step.
2. Build content to be written in the table
    a. Query data from the ‘dp’ by rule_group/rule/check names.
    b. Find desired targets, such as Vth, Idsat, etc.
    c. Find conditions, such as W/L/T/Bias, etc.
    d. Create a ‘ReportTable’ and its layout
    e. Add contents to the layout, we can do the following:
      i. Which variable/column goes first
      ii. Sort one or more columns
      iii. Filter conditions by a single value(min/max) or a list
3. Save to a file, done
    a. Specify output file path
    b. Specify output file format: xlsx or csv
    c. Whether to overwrite an existing file

In the next Chapter, we will exercise the above 3 steps by walking through examples.
