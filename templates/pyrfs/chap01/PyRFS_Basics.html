What is PyRFS

For model quality assurance and/or report, device information in tabular format is very helpful, a table can summarize a lot of information when we organize it properly. Many times, a one page table summary can beat a 30-page report full of curves, helping save a tree.

MQA has always been good at exporting reports in graphs, and also tables; however, the built-in table formats were relatively limited. Please refer to MQA user manual “Appendix -> Table Types” for all built-in tables types.

What if the built-in table formats are not sufficient?

Now, MQA 2017 release comes up with a great module called PyRFS, a Python module that allows users to fully customize (generate or update existing) tables in .csv or .xlsx files formats, using its high efficiency Python APIs.
Note: Python 2.7 based.

This tutorial will lead us to be PyRFS experts by exercising many examples.

In the first chapter, let’s firstly take a look at what benefits we get from PyRFS.

Benefits of PyRFS for MQA users

Simple enough to generate all sorts of tables quickly, while with plenty of options to customize the table in a flexibly way:
  • 1. Query and have fine control of MQA projects’ results
  • 2. Create tables in a flexible and scalable way, sort/filter/formulate/format/layout the data in many ways.
  • 3. It’s Python, so feel free to blend in your favorite python stuff.
