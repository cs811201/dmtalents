Synchronize SA and SB on Plot

The demo shows cases where we need to synchronize device instance parameters, for example, when we deal with devices with SA, SB as additional instance parameters, often SA and SB have the same length for each DUT on wafer, so that we need to view SA_SB scaling IMV plots with SA and SB synchronized.

Example MBP Script .zip file can be found at ScriptZipFiles/09_IMV_SASB_Synch.zip
Also, please use the example measurement data and model files located under Chap3/ExampleFiles/3.7/ to reproduce this demo.

When the data is loaded, we shall see SA and SB are shown on the device table, we have 3 different W/L with 3 sets of SA and SB values.


Please open MBP Script Window, let's begin with the default Idsat, we see it has an Idsat_L plot already.

If we do a "Show Data Table" on Idsat, we shall see the table contains SA and SB.


And let's go ahead to create a new Idsat_SAB plot. From Idsat, right-click, and New->Graph.


Set SA to Axis[x], and Idsat to Axis[y], as shown below.

Then "View" the plots; however, we see that SB is observed as an independent variable in the condition table, and there is only one dot on the plot, which indicated SA and SB are not synchronized on the plot and it is not what we wanted. As shown below.


In order to achieve the Synchronization, we need to switch to XY Graph type.


Now, let's put SA;SB in the Sweep1: field, and give Lable[X] SA_SB.


The Sweep1 is really the key to tell MBP to synch SA and SB on the plot, whereas Label[x] is just for customized X label display.
Now we got Idsat_SAB plot correctly, SB is on longer an independent variable in the condition table.
