Time Tasks in Flow

In Chap 2.3.2 Define a New Flow, we put together Tasks and build a flow to execute them. It would be useful if we can time our Tasks, and see how much time each Task or the entire Flow takes from beginning to finish.
This chapter shows how to do so, for example, we recorded the time when a particular Task got executed, as highlighted below.


This chapter uses the ScriptZipFiles/21_Timer.zip as an example, it is an extension of ScriptZipFiles/20_Flow.zip,
Let's use the same demo mosfet data files from $MBPHOME/demo/DC/mosfet/nmos/mdm/*.mdm.

To efficiently do so, let's create a file under flow, New -> File, as shown below, name it "util".

And, put these code into this "util", it is to define a "printNow()" method to print out the current system Data and Time, which can then be called in our Tasks.

Here is the code for copy-paste: 
import java.util.Date;
import java.text.DateFormat;
import java.text.SimpleDateFormat;

void printNow(){
	DateFormat dateFormat = new SimpleDateFormat("yyyy/MM/dd HH:mm:ss");
	Date date = new Date();
	System.out.println(dateFormat.format(date));
}
Now, we insert the following code to existing Tasks. 
flow.util::printNow();







Save the files, and re-run the flow from the beginning "OptTask1", we shall see Date and Time printed out in the log/message window.
