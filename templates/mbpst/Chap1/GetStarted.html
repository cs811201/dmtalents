Start MBP Script Window

Go for MBP menu Script -> Script Project, will pop up the Script Window; since it is a separate window, we now call the original MBP GUI the "MBP main window" to differentiate from the "Script Window".

IMV Examples

IMV is the most commonly used workforce in MBP, Device Targets such as Vth, Idlin, etc are so called IMV in MBP, and they are calculated from IV/CV data. For example, it defines scaling plots(Vth vs. Vbs, Idsat/W vs W, etc), and provides a lot help for model parameter extraction, re-targeting(re-centering) and optimization.

More examples will be demonstrated in Chap 3.

The default BSIM4 IMV settings is shown below, we could see that plenty Device Targets as well as their scaling plots have been created.


They correspond to the IMV panel in MBP main window, where all the IMV plots are displayed as long as required IV or CV curves are loaded, as shown below.


Many IMV targets can be created without any coding, MBP Script provides a lot built-in algorithm and methods to create a IMV target.
For example, Idlin. Double-click on Idlin opens the definition panel, as shown below:
Here, Idlin is defined as the Java Algorithm IMV Type, for all loaded devices, find Idlin as Current(Ids) from their Ids_Vgs_Vbs plots, where:
  • Vds=absmin(vds), meaning the absolute min Vds value from measurement data.
  • Vgs=absmin(vgs), meaning the absolute max Vgs value from measurement data.
  • Vbs=absmin(vbs), meaning the absolute min Vbs value from measurement data.

To specify the Restrictions, we could also directly assign a number, say Vgs=0; or, we could use a variable for flexibility, for example, Vgs=Vgg, where Vgg is a variable that must be defined in imv_const table, as shown below:

Save and Re-load

All the work we've done in MBP Script window can be saved as a single .zip file to a user picked folder, and that .zip file can later-on be loaded back into MBP Script window directly, no un-zip is required, which greatly facilitates our work for re-use, sharing and version control, etc.

Please use the first 3 buttons on the toolbar for Save and Load, as shown below.

When Save to current project, all the changes will reflect in the MBP main window(refresh required), for example, newly created IMV plots can be viewed and used for simulation and optimization right away.
And, besides Save it's recommended to also always dump to a .zip file, it will be easy for re-use and sharing, for example, that .zip file can be loaded into other MBP projects by all team members.
