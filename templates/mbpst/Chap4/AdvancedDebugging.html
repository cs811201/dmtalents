Advanced Debugging

MBP Script window provides advanced debugging mode, which allows us to set break-points and examine code line by line, jump into and out of a function calls, inspect variables or expressions along the way...
To trigger the advanced debugging function, we need to firstly set a few break-points.
  • To set break-points, double click on the line number.
  • To remove break-points, double click on that line number again.
Then:
  • For IMV or DP targets, we need to do Show Data Table to run the code.
  • For Flow Tasks, we click the play button to run the code.
  • For any other user defined script Files, also click the play button to run the code.
No matter which way we run the code, it will pause at the first encountered break-point, this example uses Chap3.2 Define Vth by code:
  • It firstly pauses at line 9, the line background color becomes light green.
  • Click the Step Over button, the execute moves to next line and pauses there, waiting for user to click Step Over or Step Into again.
  • On the top left, the tab next to Project is called Debug, click on it.
  • We will see Threads panel, which shows the flow of the code, from which function to which.
  • We will see Variables panel, which displays current local variables, their values will be automatically updated when we step over to corresponding lines.

  • Keep Step Over, we see the execution moves to next and next lines.
  • The local variable values update accordingly when we move on.
  • When comes to line 22, it calls another local function calculate, please click Step Into button to jump into that function.
  • If you don't wish to jump into a function, continue to click on Step Over.
  • Now, we jumped into the calculate function, continue Step Over.
  • on the left, there is an Expression tab sitting left to the Variables tab, where we can right click and Add custom variable or expressions, such as Math.abs(con-ids[index]), the value of these expressions will be evaluated on the fly.
  • This is very useful especially when we are in loops, we can inspect the variables in real time.
  • Finally, click the Continue button to run through the code to see the final output result.
  • or, click the Stop button to give up one test and make further changes in the code.
It might be a bit overkill for small applications where the simple Print can do a decent job; the advanced debugging function, however, will serve a complicate coding very well, especially when there are cascaded function calls and loops.
