Auto-run Script when loading a project

MBP Script window provides a place to auto-run code when loading the project, or loading a Script .zip file.

It is to place the code in setup node of the script tree, as shown below.
Note, init is another place MBP will execute upon loading; however, it runs even before the setup of IMVs and other stuff, so it's not recommended to put user code in init because many settings may not be ready yet.


The above screenshot shows the default code in setup for a BSIM4-Hspice project, and the code is to add two menu items to the "Extraction" menu, "GA Flow" and "NoBin Flow", as shown below. This explains that MBP's menu items are customizable, and sometimes we see different items under different projects.


So, user defined codes can be added here, and next time we open a MBP project or load another previously saved Script .zip file, these codes got executed automatically.
