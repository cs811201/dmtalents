IMV Plots

Recall that we have created an IMV target and could View Data Table, as follows, if a Table has no content, which means the Target definition's condition is not satisfied or has problem and needs correction.

Now, we could pick any column from the table to put on X or Y or Z axis to form a plot or graph. For instance, let's create the length scaling plot Idlin vs L, as shown below.


Here are the steps:
Firstly, right click on the target Idlin, and choose New -> Graph, as shown below.

Give it a name Idlin_L.

The new Graph creation panel now shows up, and by default a Simple Graph is selected, which is good for X-Y-Z Cartesian coordinates.
The Title "Idlin_L" is given the same name as the graph, now it's up to user to pick variables to put on Axis[X], Axis[Y] or Axis[P] axis, P axis just means Z axis in MBP. And, we could use the button to select a valid variable name from the dropdown list which gives all available variable names, namely column names of the Data Table.
Be default, the X and Y axes will be in Linear scale, if we want to make it Log scale, please check the Log check-box to the right.
Now we have updated Axis[X] to be L, and Axis[y] to be Idlin, and we don't need a Axis[p] in this case. As shown below.

By clicking the button, we could preview the plots, as shown below.
Variables in the table that are not assigned to X/Y/P axes will be shown in the table on the left, serving as a index table, and clicking on any row of the index table, a corresponding plot will be displayed on the right.
On the plot, the dot/symbols are measurement, and the solid lines are simulation.

Now, let's save the work in MBP Script window by clicking the second icon on the toolbar.

And, go back to MBP main window, open IMV Pages tab, as shown below. We shall see the newly defined Idlin as well as the Idlin_L plot under it.
Now, the Idlin_L plot can immediately be used for model parameter extraction or optimization.

When we create the IMV Graph, note that there is a Hierarchy field, which defines the display name in the IMV tree. If leave blank, it will inherit the Target name. However, when user specified, for instance "Length Scaling".

After Save in MBP Script window and Refresh in MBP main window for IMV Pages panel, we shall see the Idlin_L plot is organized under a tree node now called "Length Scaling"!

If we want to have multi-level tree hierarchy, please use dot sign to create the levels in the Hierarchy field, such as "Vth.bcd.def", as shown below:

Beside Simple Graph, there are many other Graph types we could use, from the dropdown list, we see:
  • Simple Graph
  • XY Graph
  • Polar Chart
  • Smith Chart
  • Histogram
  • Scatter
  • Mismatch
  • ErrorDataGrid
  • NP Corner Correlation
  • Customize

Another very useful and most commonly used Graph type is XY Graph, as shown below, which is still the Cartesian coordinates but has more attributes to use.
    Axis[y2] Display a second Y on the right, see example Chap 3.8.
    Label[X/Y/P/Y2] Customize Axis Label for display; if leave blank, the assigned variable name is used.
    Sweep 1 Synchronize multi-variable for X axis, see example Chap 3.7.
    Sweep 2 Synchronize multi-variable for P axis.
    Attached Data Attach other existing IMV data, see example Chap 3.1 and Chap 3.3.
