Specify User Python File Path

ICCAP Application Note

When we program with Python in ICCAP, it’s recommended to re-use existing python codes or files.

This application note explains a couple of ways to let ICCAP know where our external Python files are.

One way to dynamically load external Python files is shown below:


We simply append the directory to sys.path(line 2 and 3) so that Python interpreter knows where to search for files. Then, line 5 and 6 import the Python files, the format is the following:
    Import python_file_name as a_shorter_name
Here is the code for copy-paste: 
import sys, os
WorkingDir = "C:\work\ICCap\USER_PYTHON"
sys.path.append(WorkingDir)

import iccap_util as p_until
import iccap_vth as p_vth

Another way, ICCAP accepts one System Environment Variable called ICCAP_USER_PYTHON_PATH for us to specify a directory. Please set up this variable prior to starting ICCAP(both Windows and Linux), then all the .py files in that directory will be picked up when it is used.



Important note: The first time ICCAP loads an external python file(.py), the .py file got compiled to .pyc file(binary), as shown below.


And, any changes made to those external .py files after their .pyc files were created won’t refresh(re-compile) the .pyc file, unless these .pyc files were deleted.

Please refer to this note to see how to dynamically reload external python files in ICCAP.
