Add User Probe Card Pin-to-Pad Mapping

WPE FAQ

This application note explains how to add a user defined probe card Pad to Pin mapping in WaferPro Express(WPE).

Note, if we noticed “*** Switch Matrix control is Disabled.” In the Status window during launching WPE, don’t panic, it means we need to turn on the option.



To turn it on, please go to menu “Tools -> Options”, enable the “SW Matrix Control” checkbox.


Next, click the icon on the toolbar to open the “Hardware Connections” window, as shown below, and click on the “Matrix” entry in the “System hardware” section.


From the matrix driver dropdown list, we get to choose which matrix we have, this driver list is customizable via PEL or Python programming language.


After a driver is determined, we need to come to the “Probe Card Definition” tab, as shown below, by default, we may only see “DemoProbeCard” and “Pin=Pad” options here, they may have wrong Pad to Pin mapping, or wrong Pin numbers, so we must create another one.


We found that the GUI doesn’t allow us to edit the list, so to add a user defined, we need to work on the related configuration file, which are the “*.probe.card” files under $HOME/hpeesof/waferpro_version/instr directory. Where $HOME is a system environment variable.



The middle part of the *.probe.card files are displayed in WPE Hardware Connections window, so we can make a copy of the default “correspondance-DemoProbeCard.probe.card”, and rename it to “correspondance-myTestProbeCard.probe.card”, the leading “correspondance-“ and ending “.probe.card” are keywords, don’t remove or change them.

Here is how “correspondance-myTestProbeCard.probe.card” looks like where I have 16 Pins defined.


Finally, restart WPE, we will see “myTestProbeCard” shows up and ready for use.
