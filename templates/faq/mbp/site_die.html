Enable Die and Site Indices from Measurement

MBP FAQ

This note explains how to bring the Site and/or Die indices as additional identifiers in MBP, as shown below, so that we can view devices with the same geometry(W/L/T) but from different Dies/Sites as separate entries in MBP's device table.


The Site and Die indices are from the measurement data, as follows, and the two example files used here can be downloaded using these links: idvdvg_site11.mdm and idvdvg_site12.mdm .
Note: We have to use numeric value in the double quotes for all variables in the ICCAP_VALUES section, letters are not working, as shown on line 15 and 16.


Now, there are two places to update in order to enable them.

1. Add them in the Data Loader Config window, via menu "File -> Data -> Data Loader Config", as shown below.


In this case, in the mosfet tab, scroll down to the end, append Site, Die to the table, the left column is the name used in measurement data file, the right column is the name recognized by MBP which is defined in the next step in the inst.ini config file, basically we are doing a mapping here in case these names don't match.


2. Add them in the inst.ini config file, importantly this config file is model type and model level dependent, we have to edit the right one. For more details about where to find the correct one and the format of the inst.ini file, please refer to another FAQ here.


Then, restart MBP or reopen the project after we've done the above 2 changes.

We shall see Site and Die appear in the Edit Instance window by clicking on the gear icon on the left top of the device table. As shown below. Check these two check boxes, click OK, then we will see them as shown at the beginning.
