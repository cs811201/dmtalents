Turn on Unused Model Parameters

MBP FAQ

Often when we work with a model file, either MBP's default ones or user's model files, we won't have all the model parameters listed, which is totally OK because all the missing model parameters would have a default value in the simulator.

The problme comes when we want to turn a model parameter which is not initially included in the model file. For example, alpha1 , the channel length scaling parameter of impact ionization current, as shown below:

Figure 1. The impact ionization current model, cited form BSIM4 model book.

It is often found not in the model file. Only alpha0 and beta0 show up when we tried to search for it.


We could certainly edit he model file, adding "alpha1 = 0", and reload the model back in MBP via "File -> Model -> Load"; however, this application note introduces a way to quickly add it within the model table.

As shown below, at the bottom of the model table, there is a divider bar says "Unspecified Parameters", pull it up when we are in the "Rout" tab, we will see alpha1 there. Then right click on it, and do "Add to model card".
Note: All CMC standard model parameters should be found here if not used at the beginning, otherwise we will have to edit the model file manually to add it.


Now we have alpha1 in the upper part of the model table that we can work with.


As an additional note, MBP builds in explanations for every standard model parameter, we can right click on a parameter in the upper part of the model table, and open the Parameter Guide window, as shown below.


Do a "Comment" on the pop up menu, we will open the "Parameter Guide" window.
