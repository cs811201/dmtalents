Import/Export Script Settings Partially

MBP FAQ

Previously we had to Save and Load the entire Script window, so it was difficult to introduce stuff incrementally or individually. Now, we can! The new feature in MBP 2017 that allows us to import and export just part of the Script settings brings huge benefit to our daily work.

Note: A workaround was posted in MBP Script Tutorial Chapter 3.14 , that one is not recommended any more, now please start using the new feature explained in this application note.

Now, we have Import and Export on the pop-up menu in MBP Script window, wherever it applies.

To use this new feature, say in IMV, it's better to create a New -> Group, and put the targets/plots we want to export there. As shown below, we are going to create a new Group called "RMS_Grids".


As explained before, every newly created IMV item, in this case the "RMS_Grid" will be appended to the bottom of the "imv.imv" tree. Scroll down to find it, add New -> Graph under it, in this example, we created 3 ErrorDataGrid plots, as shown below. Then, right click on "RMS_Grid", and do Export.


We will be asked to save a .zip file to our harddrive, as shown below. For example, "MBP_RMS_Grids.zip" now this .zip file contains only the settings of RMS_Grids.


Next, it's ready to Import the "MBP_RMS_Grids.zip" file in another MBP session.

Say we have come to a new MBP Script window, we need to create a New Group, also name it "RMS_Grids", as shown below.


Then, right click on it, and do Import, we will be asked to pick up a .zip file, load the "MBP_RMS_Grids.zip" we just saved pleased.


Once it's loaded, we will see the 3 ErrorDataGrid plots are imported here, we may also have noticed that the "RMS_Grids" node got moved up beause now it is sorted in the alphabetic order. Done!
