Change Idlin Definition Condition

MBP FAQ

In MBP, device targets such as Idlin, Idsat, etc. are defined in MBP Script.

The default definition for Idlin is shown below, where:

    Vds = absmin(vds), Vgs = absmax(vgs), Vbs=absmin(vbs)
Which means to get the absolute minimum(absmin) value or absolute maximum(absmax) value from real-time measurement data dynamically, and use them as the condition to define Idlin, so-called Restrictions on the GUI.


However, in some cases, the absmin(vds) leads to Vds = 0 because the current measurement data contains Vds = 0 bias condition; while we still want to have Idlin defined at Vds=50mV.

One can image the fix is rather simple, by changing the Vds condtion to 50mV would do the job(Vds=0.05), it's totally ok to do so; however, we would recommend to make a copy of the default Idlin, and modify the copied one, as follows:

Right click on the existing Idlin, and do "Copy" from the popup menu.


Next, we need to click on the "imv" node(one level up), right-click, and do "Paste".


Give it a new name, such as "Idlin_vds50mv" when asked.


The newly created target would appear at the bottom of the imv list, double-click to open it, and change the Restriction accordingly. As shown below:


To verify whether the modification is valid or not, right click on the target, and do "Show Data Table".


Make sure we can see the table with correct bias conditions.


OK, one more step, is to link all the default Idlin graphs to the new target "Idlin_vds50mv".
we would go back to the origainl "Idlin", change it to "Auto Refer" type, and from the Targets drop-down list choose "Idlin_vds50mv".


Now, refresh the IMV graphes, we shall see the Idlin is with Vds=0.05.


Lastly, always remember to export the Script settings to a .zip file. Use the third icon on the toolbar.
The saved .zip file can later-on be loaded back in MBP Script window by using the first icon on the toolbar.


MBP Script window handles the .zip files directly, please do not unzip them!
