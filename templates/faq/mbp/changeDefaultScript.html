Change Model's Default Script Setting File

MBP FAQ

MBP has a default Script setting for every model, be it BSIM4, BSIMCMG, BSIMIMG or PSP, etc. where the default IMV targets, scaling plots, extraction flow, etc. are defined. For more information about MBP Script's capability(essential, powerful), please refer to MBP Script Tutorial.
As shown below:


All of these settings can be exported as a single .zip file, that can be reused or shared among other MBP users. And, users are encouraged to come up with their own programs, targets/plots definition, etc, so when that happens, end users would like to have their own Script settings/definition to be the default, so that they don't have to load a .zip every time.

Especially when the settings are standardized for a group, we want to make that default for the whole group.

Here is how we do it:

Currently, the default "script.zip" is located at the following path,
    $MBPHOME/program/dc/$simulator/$device/$model/
each model has a folder of its own, and the path/location cannot be changed yet, so what we can do is to replace the default "script.zip" file with our new version. And, importantly, the file must be named "script.zip".


For example, the "script.zip" under "bsimcmg" folder, and yes, please use all lowercase for model names.


If we don't see a folder of a particular model type in there, simply create a folder, for example "bsimimg", and put a "script.zip" file in it. Then, next time we start MBP, the new "script.zip" will be loaded as default Script setting when we switch to that model type.
