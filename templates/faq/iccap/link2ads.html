Link ICCAP to ADS Simulator

ICCAP FAQ

This note explains how to link ICCAP to ADS Simulator, hpssefosim, which is from Keysigtht's Advance Design System(ADS) software. It supports to run simulation in the following approaches:

  • ADS native mode, netlist in ADS syntax
  • Spectre mode, netlist in Spectre syntax
  • HSpice mode, netlist in HSpice syntax

    • Note: The Spectre and HSpice modes allow us to directly use foundry models in these formats with ADS simulator. As ICCAP comes with 10X paralleled ADS simulation capability, it is a big benefit in terms of simulation & optimization speed and well-supported model types.

Note: It's recommended to read the Link ICCAP to External Simulators in General firstly, including troubleshooting tips.

Entries in the $ICCAP_ROOT/iccap/lib/usersimulators config file for these 3 approaches are shown below.


ADS native mode


  1. Comfirm ADS is installed.

  2. Make sure the key system environment variable is pointed to the ADS installation.

    (1) On Windows, the variable is ADS_DIR, as highlighted in green on line 49.

    (2) On Linux, it is HPEESOF_DIR

  3. Restart ICCAP

  4. Set SIMULATOR variable in IC-CAP model to be hpeesofsim



Run Spectre model with ADS

ADS has Spectre compatibility mode built-in, by using the spmodeads as template_name as shown on line 52, which allows us to run model file in Spectre format using ADS simulator. To do so, we need to make the following change in the 4th step stated above:

  1. Set SIMULATOR variable in IC-CAP model to be spmodeads



Run Hspice model with ADS

ADS also has Hspice compatibility mode built-in, by using the hspicemodeads as template_name as shown on line 52, which allows us to run model file in Hspice format using ADS simulator. Similarly, we make the following change in the 4th step stated above:

  1. Set SIMULATOR variable in IC-CAP model to be hspicemodeads
