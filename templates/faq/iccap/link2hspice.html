Link ICCAP to HSpice Simulator

ICCAP FAQ

Verification of simulation link to HSpice

After we’ve done the above settings, to verify the link to spectre, restart ICCAP, and:

  1. load the following *.mdl file:
    …/Examples/model_files/mosfet/hnmos6.mdl

  2. Set the SIMULATOR variable to be hspice

  3. Go to the /large/idvg/ setup, open the ‘idvsvg’ plot and clear the simulated data

  4. Run a simulation and confirm that the simulated data appears on the ‘idvsvg’ plot, as shown below:

    Note: the simulation data are solid lines on the plot.


The hspice-C option

More, Hspice has a C/S mode, which stands for Client/Server. Correspondingly, in ICCAP this option is turned on by using the hspice-C option simulator link, which will give best performance when using hspice.

For example, in the usersimulators config file, define the following:

 hspice2k13C  hspice-C   c:\synopsys\Hspice_I-2013.12-SP1\WIN64\hspice.exe  ""   CANNOT_PIPE

Note: the template_name is now called hspice-C

Importantly, we also need to turn on this mode at HSPICE’s side, as follows:

  1. Start Hspui GUI, as shown below.

  2. Check the Start C/S mode and the Use C/S mode checkboxes, both!

Then, after re-starting ICCAP, we can now assign SIMULATOR to hspice2k13C to start using it.
