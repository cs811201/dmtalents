Save DUT Parameters and Variables into MDM file

ICCAP FAQ

When we do "Export Data..." from ICCAP, we don't get the DUT Parameters such as W, L, TEMP output in the .MDM file.
We got Inputs and Outputs in the Header section by defaut.


However, in case where these information are needed, such as to be used in MBP and MQA, it's always good to have them in the .MDM file.
As highlighted below.


To do this, we just need to introduce a built-in ICCAP variable called MDM_VALUES_LIST, and specify W, L, TEMP, etc. as its Value, as shown below:


This variable is categoried under "Data Manager Options". 



Specifies a space or comma separated list of Parameter or Variable names.  The values
of these Paramters/Variables are written to the MDM file.  When an MDM file with values
stored in it is imported or used to auto-create a setup, these Paramters/Variables
are atutomatically reset to the values stored in the MDM.  However, if a Parameter or
Variable no longer exists in the scope of the setup being imported to, a variable will be
created for that value in the Setup Variables Table.
Then, we do the "Export Data..." again, we will find them saved in the .MDM file.
