Specify a different Java version for Installation

ICCAP FAQ

Don’t be surprised if you found that there are many Java versions got installed on your laptop unknowingly.
Windows system and many software may have installed Java as part of their installation. The problem is that different versions of Java may have gotten installed, and sometimes it could cause trouble.

I recently experienced the following problem when I tried to install ICCAP 2018 on Win7. The InstallAnywhere progressing bar showed up, as shown below, and it disappeared right after it reached 100% and nothing else happened afterwards, a few minutes later I realized that the installation just aborted.



Later I found out it’s due to InstallAnywhere was using a system default Java which may be corrupted.

The workaround is to specify a known good Java executable to carry out the ICCAP installation in terminal, as follows: 
    [iccap_installer] LAX_VM [path_to_java_executable]



More, I learnt that this “LAX_VM” option can be applied to all Keysight EESof software installers, such as ICCAP, MBP, MQA or ADS.

Hope it solves your installation if it ran into similar problem, too.
