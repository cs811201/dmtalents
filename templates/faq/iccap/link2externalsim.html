Link ICCAP to External Simulators in General

ICCAP FAQ

This note explains how to link ICCAP to an external simulator in general.

IC-CAP enables modeling engineers to extract device model parameters using the simulator of their choice. For each model parameter extraction, measured data is compared against simulated data, and parameters are optimized to achieve the closest fit to the measured data. IC-CAP supports the following simulators' syntax:

  • spice2 (built-in)
  • spice3 (built-in)
  • hpspice (built-in)
  • hspice
  • eldo
  • smartspice
  • spectre
  • saber
  • hpeesofsim (ADS)
More, with some workaround, we can further link ICCAP to a few other SPICE simulators given their similarity to spice3. Such as:
  • LTSpice
  • PSpice

To make a link between ICCAP and some external simulator, one must edit the config file $ICCAP_ROOT/iccap/lib/usersimulators.

Figure 1.

In this config file, we can find an entry for each simulator, in the following format:

simulator_name template_name path_name host_name pipe_capability

As shown below:

Figure 2.

To link to an external simulator, the generic steps are:

  1. Install that simulator program, also make sure we have valid liense for it.
  2. Append or edit the entry in the $ICCAP_ROOT/iccap/lib/usersimulators config file for that particular simulator.
  3. Re-start ICCAP.
  4. In a model file where we want to use that simulator, specify the SIMULATOR variable to that simulator's name defined in the usersimulators.


Troubleshooting

Should you encounter any problem, please try the following, usually we can figure out the problem.
If not, please collect info from all of these places, as well as any pop-up error message, and send them back to ICCAP support team.

  1. Turn on ICCAP's Status Window, and check if any warning or error message shown there.

  2. Further, turn on the Simulation Debugger window, and check the netlist created by ICCAP and Simulator's output files.


    Note: It's recommended to close the Simulation Debugger window once the problem is addressed. Leaving it open will slow down the overall simulation speed because of additional input/output interaction.
