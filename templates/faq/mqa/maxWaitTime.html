MQA MAX WAITING TIME

MQA calls external simulators a lot, and sometimes we are in short of simulator licenses and have to wait for it.
However, by default, MQA would time out the current rule when it awaits too long, and continue to run the next rule in our QA project, which would result in failed rules.

As a workaround to this problem, we could take use of the following option supported by MQA:
  • MQA_MAX_WAITING_TIME
This is an option to be set for the OS, Windows or Linux, for example:


The value is the waiting time in seconds, up to 4 days!
The above 300 second is equally to say 5 minutes, please choose a value that is appropriate for you when necessary. =)
