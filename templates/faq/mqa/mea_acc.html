QA with Measurement - 2/4

MQA FAQ

This is part of a series of application notes, that talks about how to run MQA projects when measurement data is involved.

In the second chapter, we will see how to do accuracy check between model simulation and measurement.

Here is the output when we use the .mdm files downloaded from the first chapter, we can see model simulation as solid lines on the plot together with the measurement dots. And, RMS/MAX are calculated for each graph.


Please follow the steps explained below to reproduce the results.

Step 1. Create the MQA Rule file.
    (1) Make a copy of the auto-created Rule “meaqarxxxx.rule” and rename it to “accuracy_check.rule”.
    Note: the numbers on the auto-created Rule file name are randomly generated, we would see a different number every time MQA creates a Rule.

    (2) Make a copy of the auto-created rule index file “DataView_NMOS.index” and rename it to “rule.index”.


    (3) Edit the “rule.index”, and put “file = accuracy_check.rule” on line 2, as shown below.


    (4) Edit the “accuracy_check.rule”, turn on “simulation=yes” on line 14, 37 and 60, as shown below.




Step 2, create a new QA Project in MQA Lib Explorer.
    (1) Begin with an example BSIM4 model, which can be downloaded here, this model is in Hspice format.
    (2) Assign the “rule.index” file we just edited, as circled in red below.


    (3) And, turn to the “Measurement and ET” tab, enable the “Use measurement” checkbox and use the “Add” button to pick up all the .mdm files we downloaded from chapter one. And click “OK” button to finish the “Create QA Project” process.


    (4) We shall see the newly created “accuracy_check” project in Lib Explorer window.
    Make sure at least one of the rules checkboxes are selected, and click the green triangle “Play” button as noted Circle 1 below, it will run this project.


    (5) When the project finishes, we will see the results, as shown below, we now have RMS between the model simulation and measurement data.
    Note:
      a. MQA gives color coded RMS numbers, the range settings “pass/warning/failed” of colors can be found via menu “Options -> RMS -> Absolute/Relative RMS -> Setting…”.
      b. The “Error Plot” at the bottom left, can be turned on/off or edited via menu “Options -> RMS -> Error Plot”.



This concludes the second chapter of the QA with Measurement series using MQA.

Thank you!
