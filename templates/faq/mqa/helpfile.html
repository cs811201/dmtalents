Missing Search Tab in Help

MQA FAQ

This application note talks about an unnoticeable but very useful feature in MQA 2017’s user manual, the Search function.

We open the built-in user manual via menu “Help -> User Guide”.


We will see the Search tab in the Help window, and we can effectively search for things like “RMS”, etc.


Well, of course we take the Search function for granted; however, the reason you are guided to this note, is that this Search tab may be missing after you installed MQA 2017.

That’s most likely you have an older MQA version installed where the Search tab wasn’t enabled, and the help file cache didn’t get refreshed automatically.

So, to force the Help to be refreshed, let’s go to $HOME/AppData/Local/Keysight/Help on Windows, as shown below. We will see all(if installed) Keysight software’s Help folders, delete the old MQA folders and re-open the “Help -> User Guide” from MQA menu, it will be re-generated, and the Search tab should now show up.


This trick also applies to:
  • situations where the new version’s Help still shows old stuff, or missing(blank),
  • all other Keysight EESof software’s help contents, such as ADS, ICCAP and MBP.
clear the cache will help immediately. Thank you!
