Enable OpenOffice on Linux for Reporting

MQA FAQ

Besides PDF and HTML, MQA generates Microsoft(c) Office documents on Windows, such as Excel, PPT, and Word. The good news is, with a bit setup, we can integrate OpenOffice with MQA so as to generate Spreadsheet, Presentation and Document that are equivalent to Excel, PPT and Word respectively. Moreover, actually we can also even generate Microsoft Excel, PPT and Word on Linux, which is a feature comes with OpenOffice.

Note: There is no need to install a separate OpenOffice, the integrated version is provided in this note, as follows:

Steps:

1. download OpenOffice and configuration files from the following FTP folder:
  • ftp.eesofeda.keysight.com
  • Username: edacust
  • Password: cc39fa03de (note: the password is subject to change without notice)
  • You need to type “/files/MQA_openoffice” completely in the “Remote site” path to be able to view the files. Download all of the files.


2. Locate your MQA installation folder, for example, /home/cs/Keysight/MQA_2017
Then put these files under “/home/cs/Keysight/”, it's important that the files are at the same level of MQA installation, but not under it.
  • report.jar
  • jre1.6.0_43.tar.gz
  • oosdk.jar
  • openoffice3.3.tar.gz
  • runMeToConfigOO.sh
3. Edit “runMeToConfigOO.sh” for the correct MQA version, change to your installed version.


4. run “runMeToConfigOO.sh” and you will see a message at the end of the execution, modify PATH accordingly.

5. Now It is ok to generate report using OpenOffice in MQA.

Please go to menu "Report -> Properties", switch to Open Office, and choose file formats, as shown below:


Then, when do reporting, the Excel, PPT, Word will be in selected formats and created by by OpenOffice.


Note: OpenOffice may open a flashing window only for the first time, simply close it if it persists; later on OpenOffice will run in the background unknowingly.
