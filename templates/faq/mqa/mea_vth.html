QA with Measurement - 4/4

MQA FAQ

This is part of a series of application notes, that talks about how to run MQA projects when measurement data is involved.

In the fourth chapter, we will talk about how to extract Vth from the measurement data, as well as comparing the simulated Vth from model.

As shown below.


Because Vth is usually obtained from Ids_Vgs_Vbs curves, let’s start with editing the “Check ids_vgs_vbs_vds” rule, use the “Open the Rule” from the pop-up menu, as explained in the previous chapter.


Add line 52 – 55 in the [Target] section, and correspondingly line 59 and 60 to the end of the Rule, the CompPlot() functions will enable us to get the model simulation and measurement comparison.


The key function used here to obtain Vth is CalPage(), on line 52 and 53, we can find its detailed explanation in MQA’s user manual. Note that because we have different Vds bias conditions from the measurement data, we could actually get both Vtlin and Vtsat at the same time, in this case.

Save the Rule file, “Rerun” the highlighted Rule, as shown below.


When the project finishes, we will have two new entries in the project result tree.
(1) Vtlin vs W @ L


(2) Vtsat vs W @ L


Now that we know we could apply all kinds of functions in MQA rule, using variables/conditions from measurement as input, we can go ahead to get more device targets of interest, and plot/validate them.

This concludes the fourth chapter of the QA with Measurement series using MQA.
