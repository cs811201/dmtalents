MQA Python Post Process

Run in Command Line

The basic usage of the command line is the following: 

mpython your_scripts.py

where “mpython” is an MQA built-in command, as soon as the python code is done, we can run it within a terminal.

Note: To be able to run the command in terminals, please firstly make sure we have $MQA_HOME/bin and $MQA_HOME/tools/python/bin added to $PATH environment variable, where $MQA_HOME is where MQA is installeld, as shown below:


Then, enter the directory where the .py file resides and run the “mpython” command with the .py file name, as follows:


...


Once it's done, the new post process results are saved to user designated directory givendir, which is done the following code in this example, that saves each target's result in a subfolder of that target's name. 
def saveWithDirAndTitle(pb, newTargetName, title, yLog=False):
    dir = newTargetName + "/" + title
    yscale = "Linear"
    if yLog:
        yscale = "Log"
    pb.save(givendir=dir , Title=title, YLabel=newTargetName, YScale=yscale)
