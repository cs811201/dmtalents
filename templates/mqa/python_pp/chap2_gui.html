MQA Python Post Process

GUI Operation

Steps via GUI Operations

1. Open MQA GUI
2. Go to menu "Utilities -> Post Process"



3. In the Post-process Toolkit window, as shown below.
    (1) specify the “Source Folder” to be the IV_Sweeps example folder;
    (2) specify the “Working Folder” to be a newly created directory.



4. Now, copy the contents of “IV_Targets_basic.py” entirely to the Script area in the above window, as shown below:


And, make sure the path is correct, this app note is made out of a Windows MQA; however, it should work equally well on Linux provided proper Linux path.
srcpath = r"C:\\Users\\shuancai\\Desktop\\MQA_test\\Python_Test\\IV_Sweeps"

5. Now, click the button to run the python code.
It may take 20 some seconds to finish, and MQA GUI may seem frozen during the execution. In MQA’s Log window, at the bottom, we will see “Execute Script Done” when it’s finished.


6. As soon as the python code is finished, we will see that the new results are generated in the specified “Working Folder”, as shown below, each target and its plots are saved in a separate sub-folder.
The python code is also saved as “postprocess.py” as a backup.


7. Now, come back to the Post-process Toolkit window, and click on the “View Results” button at the lower right.
l Note: if the Post-process Toolkit window disappears, simply go to menu “Utilities  Post Process” again to bring it back, all the info are saved.


Then, the new results are loaded in MQA main GUI, we can now navigate through them, and export to reports if necessary.


8. Similarly, we can load the “IV_Targets_more.py” into the Post-process Toolkit window and play with “Freq_Sweeps” results for CV and S/Y-Parameters plots.
Note: right now MQA GUI has a 300 lines of code limit, the IV_Targets_more.py example happens to exceed this limit, so let’s run it with the command line approach, please read on. Command line execution doesn’t have such limit.
